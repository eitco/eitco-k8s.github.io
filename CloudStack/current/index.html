EITCO Cloud Stack Dokumentation

Überblick

Der EITCO Cloud Stack ist eine modulare Plattformarchitektur für die Bereitstellung von Kundenlösungen auf der Basis von Kubernetes. Wir verwenden für die Infrastruktur hauptsächlich Microsoft Azure, unterstützen aber auch andere Cloud-Anbieter und On-Premise-Umgebungen.

Durch den Einsatz von Cluster API (CAPI) und GitOps-Prinzipien ermöglichen wir eine automatisierte, konsistente und wiederholbare Bereitstellung und Verwaltung von Kubernetes-Clustern und Anwendungen.

Als Infrastruktur-Provider können verschiedene Anbieter genutzt werden, darunter:

  • Microsoft Azure

  • Amazon Web Services (AWS)

  • Google Cloud Platform (GCP)

  • Hetzner Cloud

  • Ionos Cloud

  • Vsphere

  • OpenStack

  • Proxmox

  • und weitere

Die volle Liste der unterstützten Infrastruktur-Provider findest du in der offiziellen Cluster API Dokumentation.


Die Dokumentation gliedert sich in vier Hauptbereiche:

  • Architecture: Netzwerk, Infrastruktur, Sicherheitskonzepte

  • Admin: Betrieb, Berechtigungen, Monitoring

  • DevOps: CI/CD, GitOps, Cluster-Management

  • Apps: Applikationen, Schnittstellen, Datenbanken

Zielgruppe

Diese Dokumentation richtet sich an Administrator:innen, Entwickler:innen, Architekt:innen und Projektverantwortliche innerhalb der EITCO sowie bei Kundenprojekten.

Struktur

Die Dokumentation ist modular aufgebaut. Jedes Modul kann unabhängig gelesen und erweitert werden. Die Inhalte sind ausgerichtet auf ISO 27001, DORA und internen Betriebskonzepten.

Kubernetes API Zugriff via EITCO User

Voraussetzungen

Um Zugriff auf die Kubernetes-API via EITCO User zu erhalten, müssen folgende Voraussetzungen erfüllt sein:

  • EITCO AD User

  • Mitgliedschaft in der projektspezifischen Azure-Gruppe (das Cloud CC Team fragen)

  • Installierte Tools:

    • kubectl → https://kubernetes.io/docs/tasks/tools/#kubectl

    • Pinniped CLI → https://pinniped.dev/docs/howto/install-cli/

Cluster Login

Sobald das Cluster deployt ist, erscheint eine Benachrichtigung im folgenden Microsoft Teams Kanal:

Competence Center CNCF/Cloud/AMS Monitoring - Kubernetes

Dort kann über einen Link die Kubeconfig heruntergeladen werden.

Der Download-Link kann auch über das Platform Engineering Team angefragt werden. Er setzt sich zusammen aus:
https://download-<MANAGEMENT-CLUSTER>.k8s.projekte.eitco.de/k8s/<CLUSTER>.kubeconfig

Mit der heruntergeladenen Kubeconfig kann das Cluster angesteuert werden:

export KUBECONFIG=/path/to/your/downloaded/kubeconfig
kubectl get nodes --kubeconfig my-new-cluster.kubeconfig

Dabei wird ein Fenster im Webbrowser geöffnet und man wird zur Authentifizierung über EITCO Azure weitergeleitet.

In WSL-Umgebungen wird das automatische Öffnen des Browsers nicht unterstützt. → Den angegebenen Link manuell im Browser öffnen! Siehe dazu: Webbrowser aus WSL aufrufbar machen

Wenn der Login erfolgreich war, sollte man administrativen Zugang zum Cluster haben.

Beim Safari-Browser ist es notwendig, den im Browser dargestellten Authorization Code manuell in der Shell einzutragen. → Safari unterstützt das automatische Ausfüllen nicht.

ArgoCD/Grafana Zugriff

Auch die Zugänge zu ArgoCD und Grafana werden über die EITCO AD User gesteuert. Durch die vollsändige Integration mit dem Eitco Entra ID ist es möglich, sich mit dem Eitco User per SSO anzumelden.

Das funktioniert nur, wenn bei der Bestellung des Clusters eine oder mehrere EITCO AD Gruppen angegeben wurden, die Zugriff auf ArgoCD und Grafana erhalten sollen. Ohne diese Gruppen ist kein Zugriff möglich!

Die URLs für ArgoCD und Grafana folgen dem Muster:

  • ArgoCD: https://cicd-<PROVIDER>-<CLUSTER>.k8s.projekte.eitco.de

  • Grafana: https://mon-<PROVIDER>-<CLUSTER>.k8s.projekte.eitco.de

(Beispiel: https://cicd-vsphere-cstd-alpha1.k8s.projekte.eitco.de, https://mon-vsphere-cstd-alpha1.k8s.projekte.eitco.de)
